Rollback Plan & Stakeholder Comms

Rollback Procedure — REL-003

T+0 · Trigger Condition
Error rate exceeds 1% threshold
Automated monitoring alerts fire. On-call engineer paged via PagerDuty. War room bridge opened within 5 minutes.
T+10min · Decision Point
Rollback decision by Release Manager
Incident commander assesses impact. If not resolved in 10 minutes, rollback is authorized without CAB approval per emergency protocol.
T+12min · Execution
Automated rollback script executed
Blue/green switch flips traffic back to previous stable version. DNS TTL previously set to 30s for rapid propagation.
T+20min · Validation
Health checks & smoke tests pass
Automated smoke test suite confirms baseline functionality restored. Error rate back below 0.1%.
T+30min · Post-Incident
All-clear, PIR scheduled
Stakeholders notified of resolution. Post-Incident Review (PIR) booked within 48 hours. Incident record updated in ServiceNow.
